# 0.7 — the fleet command centre (design / RFC)

**Status:** in progress on branch `0.7-ui`. This doc is the reference map for
the branch — a running version leads, the commits are the detail, this frames
the intent so a reviewer can re-orient without reading every diff.

0.7 is the UI theme: Ed Beroset's fleet-command-centre wishes, plus the
per-watch product images. It follows 0.6's clean, emoji-free dark look — so
where an older spec reached for an emoji, we use the real AsteroidOS UI icons
instead (see Icons below).

## Decisions locked with mo (Q&A, 2026-07-14)

1. **Watch-image hosting:** host-side cache + local serve — the dock fetches
   `asteroidos.org/public/img/<codename>.png` once, caches it, and serves it,
   so a LAN-only rig and the viewer's phone work without reaching the internet.
2. **No emoji.** The row-icon-strip and any new glyphs use monochrome inline
   SVGs from **asteroid-icons-ion** (mo's thin-outline Ionicons 3 fork, MIT),
   tinted with the row's theme colour via `fill:currentColor`.
3. **Image overlay** shows the product photo *and* the live screenshot; the
   dated-file screenshot stays for real captures. Full photo-compositing of the
   screenshot onto the watch face is out of scope (per-model calibration).
4. **Thumbnail** sits in its own unnamed column right of the tree glyph on
   desktop; beside the codename title on mobile.
5. **Build it, don't gate it.** beroset is a code reader waiting on delivery —
   ship working, well-scoped commits he can follow; this doc is reference, not
   a merge gate.

## Shipped on the branch

- **Watch product images.** `watchimg.py` (host cache, PNG-magic guard against
  the site's HTML-404-as-200), `watch.image` op, `/api/watch-image/<codename>`
  route, a thumbnail column, and a click-to-enlarge overlay (product photo +
  live screenshot; backdrop / × / Escape to dismiss).

## Planned pieces (each a small reviewable commit)

1. **Screen geometry + masked screenshot.** Read fb resolution
   (`/sys/class/graphics/fb0/virtual_size`) and round/square/flat-tire shape,
   and mask the overlay's live screenshot accordingly (circular for round).
   *Open:* the reliable roundness source over adb is unconfirmed
   (dconf `/lipstick/screen/*`? a machine/DeviceInfo field? a per-codename
   table as fallback?) — needs a probe on a watch that isn't mid-drain.
2. **Row-icon-strip** (asteroid-icons-ion, monochrome). Consolidate per-row
   status into a small glyph strip: wearable verdict (out of the battery
   column), watch-side charging state, a wear-trend arrow when the drain rate
   rises across tests, "last reading Xd ago", "never drain-tested". Connection
   anomalies stay in the Connection column (they're connection states).
   *Open:* which glyphs, and how far to shrink the button row underneath.
3. **Floating-overlay-ui.** One shared anchored overlay component for the Power/
   Workbench/Flashing menus and a new per-row actions list — rounded, dismiss
   on outside-click/Escape (the Control Center + this branch's image overlay
   already use the pattern; this unifies it).
4. **Fleet-stats-sheet.** Extend the Control Center into remote control of
   AOD (mcetool LPM / dconf nightstand), plus the BT/WiFi toggles it already
   has — fleet command central.
5. **cc-dconf-settings.** Watchface / wallpaper / AOD via `dconf` as ceres.
6. **wearos-backup-before-flash** (beroset). Back up a still-on-WearOS watch
   before its first AsteroidOS flash — Android-side commands, pairs with the
   0.6 data backup.

## Open questions for beroset
